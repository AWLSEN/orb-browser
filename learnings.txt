# orb-browser learnings

## Claude Code Skill (2026-04-05) — THE PRODUCT

The product is a Claude Code skill, not a web app. Users install the skill,
and their Claude Code can browse the web via orb-browser.

### How it works
1. Skill lives in skills/browser/SKILL.md with references/ and examples/
2. Claude Code auto-discovers it (shows in skill list)
3. When user asks "browse a website" or "monitor Twitter", Claude loads the skill
4. Claude uses orb-browser CLI: deploy, go, text, screenshot, sleep, wake
5. For login-required sites: Claude asks user to export cookies from Chrome
6. For scheduled tasks: Claude uses /schedule to run recurring

### Architecture
```
User's Mac (Claude Code)
  └── browser skill (SKILL.md)
      └── orb-browser CLI → Orb Cloud API → VM with Chrome
```

No backend. No Railway. No hosted app. Claude Code IS the agent.

### Cookie flow for login-required sites
1. Claude hits login wall
2. Asks user: "install EditThisCookie extension, export cookies from twitter.com"
3. User pastes JSON
4. Claude: saves to file → orb-browser login --cookies → orb-browser sleep
5. Next wake: still logged in

### Skill structure
```
skills/browser/
  SKILL.md              — triggers, core commands, patterns
  references/
    api-reference.md    — full CLI + SDK docs
    cookie-export.md    — how to get cookies from Chrome
    patterns.md         — scroll+extract, login, schedule, forms
  examples/
    browse-hn.sh        — simple browse
    twitter-watcher.sh  — login + scroll + extract
    scheduled-monitor.sh — recurring page check
```
